The Episode Six (E6) Dashboard is a web application that gives your team the ability to interact with the E6 platform as needed. The most common user personas include product managers and customer service agents. Everything available in the Dashboard is built using the same APIs with which you integrate. Dashboard user access is governed by our roles and permission framework with all activities captured in our auditable logs to ensure that you know who from your team has done what with the E6 platform. For critical actions, E6 enforces a maker-checker framework within the Dashboard.

## User personas

Any user who has been registered on the E6 platform can use the Dashboard. The Dashboard uses a roles-based access model so that roles can only view pages relevant to their definition.

For information on the roles supported by Tritium as well as each role's access to the Dashboard, see [Roles and permissions](/docs/users/roles-permissions).

## Accessing the Dashboard

Dashboard URLs will vary based on the environment you are accessing (UAT versus production, for example.) Your E6 contact will provide you with Dashboard URLs and access details prior to your first login.

## Dashboard functionality

The Dashboard is broken into functional sections, based on the level of information being managed or actions being performed.

| Section | Functionality |
| ---------- | ------------------- |
| Customer | Search for customers and access account information such as customer wallets, cards, holds, and control levels. |
| Banks | View all banks associated with your programs, add new banks, and edit each bank's currencies, programs, partners, and issuer identification numbers. |
| Programs | Manage your programs, add new programs, and update each program's control levels, fee plans, banks, partners, mail templates, and contacts. |
| Partners | View all your partners with their linked banks, programs, and partners. You can also add new partners or edit existing partner information. |
| Change orders | Manage all your change orders and approve or reject any pending changes. |
| Users | View active users, add new users, and edit user information. |
| Audit | Access and filter account activity logs by customers or users. |
